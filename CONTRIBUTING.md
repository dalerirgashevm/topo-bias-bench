# Contributing

Thanks for your interest in improving the benchmark.

## Good contributions
- add new synthetic tasks
- add stronger baselines
- add OOD / rewiring stress tests
- improve plots and reproducibility
- reduce code duplication

## Workflow
1. Open an issue describing the change.
2. Keep changes small and focused.
3. Include a short explanation of why the change matters.
4. Run the benchmark locally if the change affects results.

## Style
- prefer small, readable functions
- keep dependencies minimal
- avoid claiming expressivity gains without evidence
